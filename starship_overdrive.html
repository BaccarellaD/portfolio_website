Starship Overdrive

Java      Java FX

What It Is

Starship Overdrive is a personal project that was created to test the possibilities of JavaFX.

It is using a custom engine built on top of JavaFX which is mainly used for creating static desktop applications. Due to this there was some interesting technical challenges that had to be overcome to create a real time renderer.

What I Learned

Starship Overdrive was a really fun experiment to see how far I could push JavaFX.

I came across many problems when trying to get javafx to play nice with real time updates. I found out there is a memory leak in the library when using images within nodes, so to get around that I needed to work with object pooling.

I also implemented a 'biome' based infinite terrain generator. To do this I split the terrain up into 8 distinct tiles, then randomly generated an integer value for each one based on the value of surrounding tiles. The integer value is then used as a density value to determine how many sprites that area should contain. I then picked random spots within the tiles for each of the sprites and picked the actual sprite images based on different markov chains for each of the density regions. Finally I sorted all the sprites by Y index and placed them in the scene to avoid incorrect Z placement.
